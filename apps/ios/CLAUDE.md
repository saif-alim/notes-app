# apps/ios — SwiftUI Notes Client

**Minimal SwiftUI app.** One screen, lists notes from `services/api`, creates new ones. Shared types come from `//libs/schema:notes_swift_proto` (module `NotesSchema`).

## Layout

```
apps/ios/
├── BUILD.bazel                           # swift_library + ios_application + ios_unit_test + xcodeproj
├── Info.plist                            # ATS exception for http://127.0.0.1:3000
├── Sources/
│   ├── App/
│   │   └── NotesApp.swift                # @main, wires APIClient + root view
│   ├── Features/Notes/
│   │   ├── NotesListView.swift           # list + compose row; loading/error/a11y
│   │   └── NotesViewModel.swift          # @Observable, 4-state enum, stale-while-revalidate
│   └── Networking/
│       ├── APIClient.swift               # actor, URLSession, typed methods; env-var baseURL
│       ├── APIError.swift                # envelope decoder + error enum + userMessage
│       └── NotesAPI.swift                # Sendable protocol; enables DI + testing
├── Tests/
│   ├── NotesViewModelTests.swift         # state transitions, cache semantics, error surface
│   ├── APIClientTests.swift              # decode, URL composition, error envelope, userMessage
│   └── TestDoubles.swift                 # FakeNotesAPI, FailingNotesAPI, StubURLProtocol
└── Notes.xcodeproj/                      # generated by rules_xcodeproj (do not edit)
```

## How to run

```bash
bazel run //services/api:notes_api &    # backend on :3000
./tools/run-ios-sim.sh                  # build + install + launch on simulator
```

`run-ios-sim.sh` picks a booted simulator or boots iPhone 16 Pro. Pass a UDID to target a specific device.

## How to add a screen

See `docs/recipes/add-ios-screen.md`. Summary:

1. `Sources/Features/<Feature>/<Feature>ViewModel.swift` — `@Observable`, state enum, async `load()`.
2. `Sources/Features/<Feature>/<Feature>View.swift` — `@State` viewmodel, `.task { await vm.load() }`.
3. Wire into `NotesApp.swift` (or a future router).
4. XCTest in `Tests/` (Phase 8+).
5. Update this file — add row to **Screens** table.

## Patterns

- **State:** `@Observable` class, `@MainActor`, 4-case `State` enum: `.idle | .loading | .loaded([Note]) | .error(APIError)`. Stale-while-revalidate: keep `.loaded` rows on refresh failure; surface `lastLoadError` alert. Create failure surface via `lastCreateError` alert — draft preserved.
- **DTOs:** always import from `NotesSchema`. Do not redeclare `Note` in iOS — single source of truth is `libs/schema/notes.proto`.
- **Networking:** `NotesAPI` protocol (`Sendable`; implement fakes without touching URLSession). `APIClient` is an `actor` — safe cross-task calls, no shared mutable state. Decodes `{"error":{"code","message"}}` into `APIError.server`.
- **Timestamps:** `createdAtMs` is i64 milliseconds since Unix epoch. Convert with `Date(timeIntervalSince1970: Double(ms) / 1000)`.
- **Testing:** `FakeNotesAPI` / `FailingNotesAPI` for ViewModel tests. `StubURLProtocol` for `APIClient` decode/URL tests. `ios_unit_test` target in BUILD.bazel.

## Gotchas

- **ATS exception required.** Simulator blocks plaintext HTTP by default. `Info.plist` sets `NSAllowsLocalNetworking` so `http://127.0.0.1:3000` works. Remove when the app talks to a real TLS backend.
- **Backend must be running.** If backend is down, app shows error state + Retry button (Phase 8). Stale notes stay visible on refresh failure.
- **`baseURL` via env var.** `APIClient.defaultBaseURL` reads `NOTES_API_BASE_URL` env var; falls back to `http://127.0.0.1:3000`. Override: `NOTES_API_BASE_URL=http://host:port ./tools/run-ios-sim.sh`.
- **No reverse-conversion in schema.** Swift `NotesSchema` has Codable structs only; JSON field names mapped via `CodingKeys` (e.g. `createdAtMs` ↔ `created_at_ms`).
- **iPhone-only, simulator-only.** `families = ["iphone"]`, `target_environments = ["simulator"]`. Device support deferred.
- **`NotesViewModel.init` takes `any NotesAPI`** — pass a `FakeNotesAPI` in tests instead of the real `APIClient`.
- **`ios_unit_test` device type.** Default test runner picks "iPhone 6s Plus" which doesn't exist on iOS 26.x SDK. `.bazelrc` sets `--ios_simulator_device` + `--ios_simulator_version` so `bazel test` picks a valid device without flags.
