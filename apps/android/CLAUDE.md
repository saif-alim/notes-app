# apps/android — Jetpack Compose Android App

Notes list via platform-core UniFFI in-process (no Axum network call).

## Architecture

```
Android app (Gradle)
  └── NotesViewModel — wraps NotesCore (UniFFI-generated Kotlin object)
        └── NotesCore::createNote / listNotes
              └── platform_core.so (cross-compiled Rust, libs/platform-core)
                    └── InMemoryNotesStore (same Rust impl as the Axum backend)
```

The shared-core story: iOS talks to Axum over HTTP; Android calls
platform-core directly in-process. Both use the same `InMemoryNotesStore`
impl — proves portability without duplicating the REST wire.

```
apps/android/

├── build.gradle.kts
├── gradle.properties

    └── src/androidTest/          # (future: instrumented smoke test)
```

## How to run

### Prerequisites

```bash
cargo install cargo-ndk            # cross-compile tool
# Install Android NDK via Android Studio or:
#   sdkmanager "ndk;26.3.11579264"
export ANDROID_NDK_HOME=/path/to/ndk

# Install Gradle wrapper (run once from apps/android/):
#   gradle wrapper --gradle-version 8.10.2
```

### Build native library

```bash
./tools/build-android.sh
# Writes arm64-v8a/libplatform_core.so + x86_64/libplatform_core.so
# to apps/android/app/src/main/jniLibs/
```

### Run the app

```bash
cd apps/android
./gradlew installDebug
adb shell am start -n com.notes/.MainActivity
```

Or set `SKIP_NATIVE_BUILD=true` to skip the pre-build cargo step if `.so` files
are already present.

## Gotchas

- `jniLibs/` is gitignored. Run `build-android.sh` before `./gradlew installDebug`.
- `uniffi/platform_core/platform_core.kt` is committed (generated from the macOS
  dylib via `uniffi-bindgen`; regenerate if FFI surface changes).
- JNA (`net.java.dev.jna:jna:5.14.0@aar`) is required — it's how the uniffi-generated
  Kotlin code calls into the native library.
- MinSdk 26 — required for modern Kotlin coroutines and StateFlow.

## How to update the FFI surface

If you change `libs/platform-core/src/ffi.rs`:
1. Rebuild the dylib: `cargo build -p platform_core --features ffi --release`
2. Regenerate bindings: `cargo run --manifest-path tools/uniffi-bindgen/Cargo.toml -- generate --language kotlin --library target/release/libplatform_core.dylib --out-dir apps/android/app/src/main/java`
3. Commit the updated `platform_core.kt`.
4. Rebuild native .so: `./tools/build-android.sh`
